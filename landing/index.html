The Astro framework that survives forks.

Quick start

Five commands from clone to deployed.

1 — Clone
git clone https://github.com/Mumega-com/inkwell my-site
cd my-site && npm install
2 — Configure
# Edit inkwell.config.ts:
#   name, domain, tagline, theme, features, plugins
3 — Develop
npm run dev          # Astro on :4321
cd workers/inkwell-api && npx wrangler dev   # Worker on :8787
4 — Provision Cloudflare
npx wrangler d1 create inkwell-core
npx wrangler kv namespace create CONTENT
# Update workers/inkwell-api/wrangler.toml with the IDs
5 — Deploy
npm run deploy       # Build + ship to Cloudflare Pages

Why fork instead of build?

  • Microkernel + plugins. 5-file kernel, ~430 lines. Plugins are self-contained verticals you can remove from config.plugins[] and they vanish from the build.
  • One config file. inkwell.config.ts drives theme, features, plugin set, adapters. Toggle a feature off — its routes and UI disappear. Change an adapter — infrastructure swaps with zero code changes.
  • Cloudflare-native. Workers, Pages, D1, KV, R2, Workers AI. Free tier handles real production load.
  • Agent-operable. MCP server with 16+ tools so your AI agent can publish content, manage commerce, run analytics, upload media. Works with Claude, ChatGPT, Cursor, any MCP client.
  • Designed to survive forks. The kernel evolves; your customizations stay in your fork. Workspace-pattern proposal in flight (#53) to make this even cleaner.

What's inside

Kernel

Microkernel + ports

5 files, ~430 lines. Plugin manifests, hexagonal ports, RBAC, theme. Swap infrastructure (D1 → Postgres) by changing one line.

Plugins

17+ vertical modules

auth, content, commerce, courses, contracts, analytics, mcp, dashboard, telegram, organism, sync, media, more.

Content

MDX + knowledge graph

[[wikilinks]] auto-build a graph. KaTeX math. 14 custom blocks. Pagefind search. Multi-language with hreflang.

SEO

JSON-LD + Open Graph

14 schema.org types. Sitemap, RSS, llms.txt. Google Scholar / Dublin Core via ScholarMeta for academic forks.

Agents

MCP server, 16+ tools

publish_content, get_dashboard, create_checkout, upload_media, generate_image, more. Connect any MCP client.
