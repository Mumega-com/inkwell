## 2025-02-12 - Decouple Canvas State from React Renders and Pause Animations When Off-Screen
**Learning:** We observed that updating hover state in React (via `useState`) for deeply nested or complex canvas elements like `KnowledgeGraph` triggers a re-render for the entire component containing the canvas, and frequently restarts animation loops, leading to significant unnecessary overhead. Additionally, continuous animation loops (`requestAnimationFrame`) in `NetworkHero` run indefinitely even when completely off-screen, eating CPU and battery life unnecessarily.
**Action:** Use `useRef` to maintain canvas-specific interactive state (e.g., `hoveredNodeRef`) and imperatively update DOM properties (`canvas.style.cursor`) to avoid React re-renders while animating. Combine this with `IntersectionObserver` to pause drawing/simulating operations when the visual is not intersecting with the viewport.
