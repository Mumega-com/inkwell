## 2026-04-24 - Optimize KnowledgeGraph Simulation
**Learning:** In continuous physics simulations (like `KnowledgeGraph.tsx`) running inside `requestAnimationFrame`, nested loops evaluating node interactions (like forces or repulsion) can become severe performance bottlenecks if not handled properly. An `O(N²)` nested loop that evaluates `nodeA` vs `nodeB` and then redundantly evaluates `nodeB` vs `nodeA` causes excessive CPU usage. This pattern differs from `NetworkHero.tsx` which avoids evaluating node relationships twice.
**Action:** When implementing custom N-body physics or interaction loops, always optimize distance/force evaluations from `N(N-1)` to `N(N-1)/2` by iterating pairs (e.g., `for (i=0...N) for (j=i+1...N)`) and applying calculations to both items. This simple structural shift significantly lightens the main thread overhead for continuous animations.
